unicode，utf-16,utf-32,utf-8介绍

unicode

Unicode(UCS)编码长度是31位，可以表示$2^{31}$个字符。UCS-4根据最高位为0的最高字节分成$2^7$=128个group。每个group再根据次高字节分为256个平面（plane）。每个平面根据第3个字节分为256行 （row），每行有256个码位（cell）。group 0的平面0被称作BMP（Basic Multilingual Plane）。如果UCS-4的前两个字节为全零，那么将UCS-4的BMP去掉前面的两个零字节就得到了UCS-2。每个平面有$2^{16}$=65536个码位。

用比较简单的话来说就是，Unicode定义了所有可以用来表示字符的数值集合（称之为Code Point）。UTF-8和UTF-16等UTF标准定义了这些数值和字符的映射关系。

utf-16 vs bmp vs ucs-2

bmp等同于ucs-2用两个字节来表示，形式如:0x__
utf-16 当 编码<0x10000时，和bmp是一样的，当编码大于那个值时，因为bmp中有一些控制字符，可以用这些字符来扩展字符的编码，故utf-16可以看成是bmp的父集

**utf-32 vs ucs-4”

utf-32和ucs-4也是一样的，是用4个字节表示的，在unix下的宽字符就是用4字节表示，形式如：0x00000041

utf-8

utf-8是不像utf-32,或者utf-16是定长的，它是不定长的，由1-6个字节组成
0-127的ascii编码的字符与其的utf-8编码相同。其他有由下编码表：

ascii vs latin-1

ASCII-American Standard Code for Information Interchange

  • Character repertoire:
    • ASCII 严格来讲就是7个bit大小的字符集，也就是code point介于0-127之间的字符集合。
  • Character code:
    • 32-126之间的字元是可打印字元，其他是控制字元。
  • Character encoding:
    • ASCII可以没有任何编码就可以在计算机中用一个字节表示，也就是每个code point被表示成等价的单字节二进制形式。

LATIN1-8bit sing byte coded graphic character sets.(别名：ISO-8859-1)

  • Character repertoire:

    • Aka ISO-8859-1。是7bit ASCII 字元集的扩充的一种，是8个bit大小的字符集，也就是code point 介于0-255之间的字符集和。

  • Character code:

    • 包含有191个可打印字符，其余是控制字符或者扩展的欧洲特殊字符。
  • Character encoding:
    • 与ASCII相似， 每个code point被表示成等价的单字节二进制形式。
